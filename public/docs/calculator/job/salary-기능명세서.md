## 2. 화면 위치

2. **메타데이터**: '연봉 계산기 · 월급 실수령액 계산기 | 2025 최신 세금 및 공제액 계산 - JIKO 계산기' Title 적용

## 4. 화면 기능
1. 입력 데이터
ㄴ 연봉/월급 선택
ㄴ 연봉/월급 입력(원) : 직접입력, (단위 버튼: 100만, 300만, 500만, 1000만, 3000만, 5000만, 1억 등 제공)
ㄴ 퇴직금 : 별도, 포함
ㄴ 비과세액(식대포함) : 기본 20만원, 직접입력
ㄴ 부양가족수(본인포함) : - 1명 + 형태로 가감
ㄴ 20세 이하 자녀수 : - 0명 + 형태로 가감

2. 출력 데이터
ㄴ 예상 소득액(월) : 공제 전 금액
ㄴ 공제액 합계
ㄴㄴ 공제항목 및 각 금액 : 국민연금, 건강보험, 정기요양, 고용보험, 소득세, 지방소득세
ㄴ 예상 실수령액(월) : 공제 후 금액
ㄴ 회사 부담액
ㄴ 회사 부담액 합계
ㄴㄴ 회사 부담액 항목 및 각 금액 : 국민연금, 건강보험, 정기요양, 고용보험

3. 버튼 기능
ㄴ **초기화 및 계산하기** : 공통 컴포넌트인 **`CalculatorButtons`**를 적용하여 UI 인터랙션(활성 상태, 호버 효과 등)을 전역 표준으로 통일했습니다. 
ㄴ **결과 복사 & 공유하기** : 코어 컴포넌트인 **`CalculatorActions`**를 배치하여 결과 데이터를 텍스트 형식으로 복사하거나 카카오톡으로 공유할 수 있는 기능을 제공합니다.
ㄴ **결과 자동 스크롤**: 계산 완료 시 결과 영역으로 부드러운 스크롤 이동 (`useCalculatorScroll` Hook 사용).

4. UI/UX 기능
ㄴ 초기화 버튼 클릭 시 흔들림 애니메이션 (shake) 효과 적용
ㄴ **표준 검증 UI**: 필수 항목 누락 시 빨간색 테두리(Ring) 및 흔들림 효과, 하단 에러 메시지 노출 (Pulse 애니메이션)

5. 하단 정보 영역 (SEO 및 사용자 가이드 - page.tsx)
ㄴ 검색 엔진 최적화(SEO)를 위한 JSON-LD 라우팅 보강
ㄴㄴ `SoftwareApplication` 및 `FAQPage` 외 추가 범용 `schema` 스크립트 작성 및 로드
ㄴㄴ 구글 서치 콘솔(GSC) 등록 완료 (루트 경로 HTML 소유권 확인)
ㄴㄴ `sitemap.ts` 및 `robots.ts`를 통한 동적 크롤링 및 인덱싱 허용 (검색 노출 최적화)
ㄴ UI 디자인 : 본체와 동일한 카드형 레이아웃 및 다크모드 대응 적용
ㄴ 구성요소
ㄴㄴ 소개(H1) : 연봉/월급 실수령액 계산기 및 차이점 안내 (연봉 계산기, 월급 계산기 키워드 모두 포함)
ㄴㄴ 사용 방법 : 연봉/월급, 퇴직금, 비과세액, 부양가족수를 통한 세금 및 실수령액 계산 가이드
ㄴㄴ 계산 예시 : 30세 남성, 연봉 5000만원 기준 예상 실수령액(월) 도출 예제
ㄴㄴ FAQ : 연봉 계산기와 월급 계산기의 차이점, 2025년 기준 4대보험 공제율 변경점 등 유사 키워드 포함 기획 (**공통 `FAQ` 컴포넌트 적용**)
ㄴㄴ 직장 계산기 더 보기 : 다른 직장 계산기 메뉴 바로가기 (**공통 `JobMoreCalculators` 컴포넌트 적용**)

## 5. 계산 로직 (2025년 기준)
1. **과세 대상액 산출**: `(연봉 혹은 월급) - 비과세액(식대 등)`
   - 퇴직금 포함 시: `총액 / 13`을 퇴직금으로 산정하고, 나머지 `12/13`를 연봉으로 간주
2. **사회보험료 공제 (근로자 부담분)**:
   - **국민연금**: 과세대상액의 4.5% (기준소득월액 상한액 적용)
   - **건강보험**: 과세대상액의 3.545%
   - **장기요양보험**: 건강보험료의 12.95%
   - **고용보험**: 과세대상액의 0.9%
3. **근로소득세 및 지방소득세**:
   - **소득세**: 국세청 간이세액표 기반 (부양가족수, 20세 이하 자녀수 반영)
   - **지방소득세**: 소득세의 10%
4. **예상 실수령액**: `월 평균 소득액 - 소득세 - 지방소득세 - 사회보험료 합계`
5. **회사 부담액**: 국민연금 4.5%, 건강보험 3.545%, 장기요양 12.95%, 고용보험 1.2% (기준), 산재보험(업종별) 등

## 6. 예외 처리

- 프로젝트 공통 CSS (tailwind) 및 JIKO-calculator.md 디자인 패턴 준수 (카드형 UI, 다크모드 적용).
