# 환율 계산기 기능 명세서 (Exchange Rate Calculator)

## 1. 개요
* **목적**: 다양한 국가의 통화 가치를 실시간 환율 기준으로 신속하고 정확하게 변환하여 투자, 여행, 쇼핑 시 환전에 드는 시간과 노력을 절약합니다.
* **경로**: `/calculator/finance/exchange-rate`
* **UI/UX 원칙**: JIKO 표준 (카드형 레이아웃, `max-w-3xl`, Smooth Scroll, 공통 하단 컴포넌트)

## 2. 화면 위치 및 파일 구조
1. **경로**: `/calculator/finance/exchange-rate`
2. **파일 구조**:
   - 서버 : `app/calculator/finance/exchange-rate/page.tsx`
   - 클라이언트 : `app/calculator/finance/exchange-rate/ExchangeRate.tsx`
   - 수출입은행 API : `app/api/exchange-rate/koreaexim/route.ts`
   - ExchangeRate-API : `app/api/exchange-rate/exchangerate/route.ts`

## 3. 계층 구조 및 SEO (Common Architecture)
1. **Breadcrumb**: `홈 > 계산기 홈 > 금융 계산기 > 환율 계산기`
2. **JSON-LD**: `SoftwareApplication` 및 `FAQPage` 스키마 적용.

## 4. JIKO만의 차별화 기능 (Differentiation)
1. **2 in 1 실시간 다중 모드 탭(Tab) 연산**: 
   - JIKO 표준 와이어프레임 1.3 탭(Tab) 구조를 채택하여, `[💱 환전 계산]`과 `[📈 환율 시뮬레이션]`을 분리해 복잡도를 낮추고 모바일 사용성을 극대화합니다.
2. **환전 수수료 우대율(%) 적용 기능 (한국 유저 특화)**: 
   - '환율 우대 90%', '80%' 등 은행 앱에서 제공하는 실제 수수료 할인 개념을 도입합니다.
   - 매매기준율과 현찰살때의 차액(스프레드)에서 사용자가 설정한 우대율만큼 할인해주는 정확한 은행권 실무 로직을 지원합니다.
3. **지능적인 API 하이브리드 로직 & 브라우저 실시간 계산**: 
   - 타이핑할 때마다 API를 호출하는 대신, 초기 1회(또는 통화 변경 시)에만 환율을 가져오고 사용자가 금액을 입력할 때는 클라이언트(브라우저)에서 즉각적으로 연산하여 API 한도 초과를 완벽히 방지합니다.
   - 원화(KRW) 기반의 은행 환율은 수출입은행을, 달러-유로 같은 이종 통화 변환이나 수출입은행 장애 시에는 ExchangeRate-API를 사용하는 하이브리드 라우팅을 구성합니다.

## 5. 화면 기능
### 5.1 입력 데이터 (JIKO 1.3 탭 구조 활용)
상단 탭을 통해 두 가지 모드를 전환합니다.

1) **[💱 환전 계산 모드] 탭**: 송금액을 입력하면 수취액이, 수취액을 입력하면 송금액이 즉시 역산됩니다.
- **통화 선택**: 원화 ↔ 외화, 외화 ↔ 외화 지원
- **환율 기준 선택**: 매매기준율, 현찰(살때/팔때), 송금(보낼때/받을때)
- **환율 우대율 선택**: 90%, 80%, 50%, 없음 (실제 은행 앱과 동일한 할인 적용)

2) **[📈 시뮬레이션 모드] 탭**: 사용자가 원하는 가상의 환율을 설정하여 미래의 환전 시나리오를 계산합니다.
- 타겟 환율 직접 입력 또는 현재 환율 대비 증감률(%) 입력 기능 제공.

### 5.2 출력 데이터
각 탭에서 입력 즉시 하단에 직관적인 결과 표시.
1) **[환전 계산 모드]**: 
   - 우대율이 적용된 최종 환율 정보 (1 단위 당)
   - 보내는 금액 [ A ]원은 [ B ]나라 통화로 **[최종 결과값]** 입니다.
2) **[시뮬레이션 모드]**: 
   - 환율이 [ A ]% [상승/하락]하면, 기존 환전 대비 **[차액 결과값]** 차이가 납니다.

### 5.3 버튼 및 UX
- **CalculatorButtons**: 표준 초기화/계산하기 버튼 적용.
- **오류 UX**: 필수값 미입력 시 JIKO 최신 표준인 "🚨 필수 항목을 모두 입력해주세요" 펄스 애니메이션(화면 흔들림 제거) 적용.

- **아이콘 및 제목**: 🌏 환율 계산기 가이드 (원화-외화 실시간 자동 변환)
- **내용**: 해외여행, 직구, 유학, 해외 주식 투자 시 필수적인 환전 계산을 위한 전문가용 계산기입니다. 달러, 엔화, 유로 등 주요국 통화의 실시간 환율을 적용하여 1원부터 역산까지 한 번에 계산됩니다.

### 1.7 사용 방법 및 계산 예시 (2단 그리드)
- **💡 사용 방법 (좌측)**: 
  - 환전할 금액과 통화를 입력하면 실시간 환율을 적용한 결과를 즉시 확인할 수 있습니다.
  - 시뮬레이션 모드에서는 미래의 환율 변동에 따른 예상 금액을 미리 계산해볼 수 있습니다.
- **📊 계산 예시 (우측)**: 
  - (환전) 100만원을 미국 달러로 환전하면? -> 약 730달러!
  - (시뮬레이션) 만약 환율이 10% 오르면, 100만원은 약 660달러!

### 1.8 환율 및 수수료 상식 (2단 그리드 - 추가 카드 세션 2개)
- **카드 1: 환율 우대 90%의 진짜 의미는?**
  - 은행이 이윤을 남기는 현찰 살 때 환율과 매매기준율의 차이(스프레드 마진) 중 90%를 할인해준다는 의미입니다.
- **카드 2: 이중환전(DCC) 주의사항**
  - 해외에서 신용카드 결제 시 현지 통화가 아닌 원화(KRW)로 결제할 경우 3~8%의 추가 수수료가 발생할 수 있으니 주의하세요.

### 1.9 자주 묻는 질문 (FAQ)
- **Q. 주말이나 공휴일에도 실시간 환율이 반영되나요?**
  - **A.** 네, KOREA EXIM(수출입은행) 데이터의 경우 직전 영업일의 최종 고시환율을 적용하며, 이종 통화 간 변환 시에는 글로벌 실시간 환율 API를 통해 365일 언제나 가장 정확한 환율을 제공합니다.
- **Q. 매매기준율과 현찰 살때 환율의 차이가 무엇인가요?**
  - **A.** '매매기준율'은 수수료가 포함되지 않은 도매가 환율입니다. 반면 우리가 은행에서 실제 지폐를 바꿀 때는 외화 운송 및 보관 비용(스프레드 마진)이 포함된 '현찰 살때/팔때' 환율이 적용됩니다. 본 계산기에서는 두 가지 기준을 모두 지원합니다!
- **Q. 환율 우대 90%를 적용하면 얼마나 싸지는 건가요?**
  - **A.** 은행이 남기는 마진(현찰 살때 환율 - 매매기준율)의 90%를 깎아준다는 의미입니다. JIKO 계산기에서 우대율 옵션을 선택하시면 은행 앱과 동일하게 수수료가 절감된 최종 금액을 즉시 확인하실 수 있습니다.

### 10. 개발 방법
#### API 통신 한계점 보완 및 역할 분리
1. **ExchangeRate-API (초고속 실시간 데이터 & 이종 통화 전담)**
   - 달러(USD) -> 유로(EUR) 등 원화가 끼지 않은 이종 통화 간 계산 시 주력으로 사용합니다.
   - 1시간 단위 서버 캐싱(`revalidate: 3600`)을 적용하여 불필요한 비용과 호출 한도를 방어합니다.
   - 한국수출입은행 API 오류 시 메인 Fallback으로 작동합니다.

2. **한국수출입은행 (정밀한 은행권 스프레드 및 우대율 계산 전담)**
   - 원화(KRW) 기반의 현찰 사고팔때, 송금 보낼때받을때 등 은행 마진이 포함된 기준 환율 제공 시 활용합니다.
   - **주말/공휴일 대응(중요)**: 수출입은행 API는 비영업일에 최신 데이터를 응답하지 않으므로, 백엔드(`route.ts`)에서 오늘 데이터가 비어있을 경우 재귀적으로 어제, 그제(금요일) 등 가장 가까운 직전 영업일 날짜로 재요청하여 무조건 값을 받아오도록 구성해야 합니다.

3. **Client-side 연산 최적화 로직**
   - **`onChange` 시 API 호출 절대 금지**: 환율 데이터는 초기 렌더링 시나 통화가 변경될 때 딱 한 번만 Fetch하여 상태에 저장합니다. 사용자가 키보드로 100,000을 연달아 입력할 때는 브라우저 메모리에 저장된 기준 환율로만 연산하여 API 낭비를 원천 차단합니다.
